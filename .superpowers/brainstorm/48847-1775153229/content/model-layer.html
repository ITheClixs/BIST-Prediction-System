Model Details

XGBoost Best-in-class for tabular data. Two heads: classification (direction) + regression (% move). Trained per-sector or globally with stock embeddings. Provides feature importance for explainability.
LightGBM Faster training, handles categorical features natively (sector, day-of-week). Slightly different splitting strategy than XGBoost — diversity improves ensemble. Same two-head approach.
LSTM Sequence model — feeds 30-day windows of features. Captures temporal dependencies that tree models miss (e.g., momentum shifts, mean reversion timing). PyTorch.
Transformer Self-attention over 60-day windows. Better at long-range dependencies than LSTM. Can attend to specific past events (earnings, macro shocks). PyTorch.
Ensemble Meta-learner (logistic regression or small XGBoost) trained on validation set predictions. Learns optimal weighting. Can override per-stock if one model dominates for certain tickers.

Training Strategy

  • Walk-forward validation — train on past, validate on next N days, slide window forward. No future leakage.
  • Retrain cadence — full retrain monthly, incremental update weekly
  • Lookback — 3-5 years of history for tree models, 1-2 years for deep learning
  • Target — dual: next-day direction (binary) + next-day return (regression)
  • Model registry — version each trained model, track performance over time, auto-rollback if new model underperforms

Confidence Calibration

  • Platt scaling — fits a sigmoid to map raw ensemble scores to calibrated probabilities
  • Reliability — when the model says "78% UP", historically ~78% of such predictions were correct
  • Threshold — configurable minimum confidence to surface a signal (default: 60%)
