Architecture Approaches

Pick the architecture that best fits your goals

A

Monolith Pipeline

Single Python package. Data fetching → feature engineering → model training → prediction all in one linear pipeline. Rust used for a fast technical indicator calculation library (called via PyO3).

Pros

  • Simplest to build and debug
  • Single entry point, easy CLI
  • Fast to get first predictions

Cons

  • Hard to swap/test components independently
  • Retraining requires running entire pipeline
  • Doesn't scale well as features grow
B

Modular Pipeline (Recommended)

Separate modules for data ingestion, feature store, model training, and prediction. Each module has clear interfaces. Rust powers a high-perf feature computation engine. SQLite for local data/feature storage.

Pros

  • Test and iterate each component independently
  • Add new data sources or models without rewiring
  • Feature store enables fast experimentation
  • Balanced complexity vs. flexibility

Cons

  • More upfront design work
  • Slightly more files/structure
C

Microservice / Event-Driven

Each component runs as an independent process communicating via message queues (e.g., Redis). Data collectors, feature engine, model servers all separate. Rust as standalone services.

Pros

  • Maximum scalability and isolation
  • Can run data collection 24/7 independently
  • Production-grade architecture

Cons

  • Massive overkill for a CLI tool
  • Operational complexity (Docker, queues, etc.)
  • Slowest to build and debug
