Why Split Rust / Python?

  • Rust — pure numerical computation: indicators, correlations, pattern matching. These are hot loops over large arrays. 10-50x speedup over numpy/pandas.
  • Python — NLP sentiment scoring, API interactions, calendar/temporal logic. These are I/O-bound or need Python NLP libraries (transformers).

Feature Store Design

  • Keyed by (ticker, date)
  • Incremental computation — only recalculate for new dates
  • Feature versioning — schema migrations when adding new features
  • Missing value strategy: forward-fill for macro, NaN flag for sentiment gaps
